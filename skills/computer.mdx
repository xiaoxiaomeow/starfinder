# 计算机（智；训）

:::origin[[英文原文](https://www.aonsrd.com/Skills.aspx?ItemName=Computers)]
:::

你可以操作、操控并入侵计算机系统。若无法直接接触计算机系统的UI，则必须使用黑客工具包来访问和操控该系统。计算机的详细规则从第213页开始。

计算机系统通常会为授权用户提供"root权限"，允许他们以标准动作访问任何信息或功能，且无需进行计算机检定。防火墙可以隔离计算机的特定区域，并为不同用户授予这些区域的root权限。

计算机技能多数任务的基础DC为13+(4x计算机层级)。GM可根据实际情况调整这些DC。

## 访问未加密系统

你可以使用计算机技能访问未加密系统的基本功能。对于没有防护措施或防火墙的公共计算机，访问其信息或功能的DC通常为10。与计算机技能其他任务不同，若你在此任务上取20（耗时2分钟），即使未受训也可尝试访问系统。加密系统或未加密系统中的加密区域需要通过[入侵系统](#入侵系统)任务来访问。

## 制造计算机

若拥有足够的计算机技能等级，你可以制造计算机。造物规则见第235页。

## 伪造或识破伪造文件

你可以使用计算机技能伪造官方文件，耗时1d4分钟。GM会秘密进行伪造的计算机检定，因此你无法确定伪造效果。该检定将与审查文件真实性者的计算机检定对抗。

识破伪造的DC取决于文件类型及GM判定的其他情况，如下表所示。

:::table
| 情况 | DC调整值 |
| :---- | :----: |
| 文件与已知信息或命令矛盾 | -2 |
| 审查者非常熟悉此类文件 | -2 |
| 审查者不熟悉此类文件 | +2 |
| 审查者仅粗略检查文件 | +2 |
| 伪造者持有类似文件样本 | +8 |
:::

## 破坏或修复系统或模块

你可以花费每计算机层级10分钟的时间，使用计算机技能修复被停用的系统或模块，或破坏已移除或停用的系统或模块。DC取决于计算机层级。若修复失败且差值至少为5，将意外摧毁该模块或系统。破坏或修复计算机系统时不可取20。

## 识破虚假终端

若你仅有计算机访问权限而无root权限，可能实际访问的是虚假终端（该反制措施会提供虚假文件）。成功通过检定可察觉虚假终端，DC取决于计算机层级。

## 停用或操控模块

拥有root权限者可通过DC10的计算机检定以标准动作停用或操控反制措施或模块。仅有访问权限者也可尝试此类检定，但需先用[入侵系统](#入侵系统)入侵防火墙区域。激活或停用模块通常为标准动作，安装或移除模块通常耗时每层级1分钟。这些任务的DC等同于[入侵系统](#入侵系统)的DC，且不可取20。

## 获取root权限

已有计算机访问权限者可尝试提升授权等级。其流程类似[入侵系统](#入侵系统)，但DC提高20。许多情况下无法获取root权限，所有超出基本功能的操作都需单独进行破坏、修复、停用、操控模块检定。若防火墙分割了有独立的root权限用户的区域，需分别获取权限。获得root权限后，可通过成功的[入侵系统](#入侵系统)检定来修改其他用户的root权限。

## 入侵系统

你可以对无法访问的计算机系统进行入侵，通常耗时每层级一个整轮动作。每使检定DC提高5点，耗时即可减半（最低一个整轮动作）。成功后可以访问没有被防火墙阻挡的区域（第217页），使用基本功能并进行需访问权限的后续检定。每个防火墙区域需额外检定。

入侵失败可能触发已安装的反制措施。若在未停用反制措施时在入侵系统上取20，将自动触发所有反制措施。
